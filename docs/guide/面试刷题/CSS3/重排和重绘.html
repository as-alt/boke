# 页面生成的过程

  1. HTML被HTML解析器解析成DOM树
  2. css样式被css解析器解析成Style树
  3. 结合DOM树和Style树生成渲染树(render tree),此过程被称为Attachment(连接)
  4. 生成布局(flow),浏览器在屏幕上画出渲染树中的所有节点
  5. 将布局绘制(paint)在屏幕上,显示出整个页面

第四步和第五步合起来就是渲染,也是最耗时的部分

# 谁会影响谁?

  • 重绘：元素的外观被修改
  • 重排：重新生成布局，重新排列元素

如果单单修改元素的外观，肯定不会引起页面重新布局,但当浏览器完成重排后，将会重新绘制此次重排影响的元素，比如改变元素的高度，元素甚至周边的DOM都需要重新绘制

所以 重绘不一定导致重排，但重排一定导致重绘

# 重排

当DOM的变化影响了元素的位置或大小,浏览器需要重新计算元素的属性,将其安放在界面正确的位置，这个过程叫重排

简单来说就是重新生成布局,重新排列元素

# 重绘

当一个元素的外观发生改变但未影响布局，重新把元素外观绘制出来的过程，叫重绘
